# goksung 채점 루브릭 — 세부 항목 그물식(투망식) 통합본

> 5개 대분류(문제정의20/해결과정20/결과물완성도25/업무적용가능성25/사례공유력10) 각각을 전문가 관점에서 세분화한 결과를 통합. 작업은 team 5인(worker-1~5, Opus)이 병렬 수행, 이 문서에서 team-lead가 취합·정리했다. 원본 개별 파일: `.omc/drafts/rubric-detail-{problem-definition,solution-process,output-completeness,applicability,shareability}.md`.
>
> **총 세부 항목 수**: 56개 (12+11+10+11+12) — 목표(대분류당 8~12개)를 모두 충족.
> **신규 필수 후속질문 순증**: 기존 합의안(`goksung-write-post-option.md`) 대비 **딱 2개** ("기존 대안이 왜 안 됐나" / "왜 이 방법을 골랐나") — 그 외 모든 세분화는 게이트/생성규칙/템플릿 힌트로 흡수해 인터뷰 저마찰 원칙을 지켰다.

**최종 A그룹**: 정량 후속(기존) + 대안 부재 후속(신규) = **2개**. 나머지 10개는 코어 질문/🎯섹션으로 이미 커버되거나 수동 확인(B).

## SKILL.md 반영 제안 — 2차 보완 반영분 (1차 제안에 얹는 델타)

**편집 3 (Phase 3 후속질문) 델타:**
1. **신규 3번째 후속질문 (S13, 권장)**: Q4(막혔던 부분) 또는 Q5(결과 임팩트)에 부착 — `"AI가 만든 결과물이 정확한지 어떻게 확인하셨나요? (예: 샘플 대조, 수동 검산, 테스트)"`. 근거: AI 리터러시 감별 최상위 신호이면서 기존 76개 항목 중 유일하게 "질문 없이는 답이 안 나오는" 사각지대. 한 문장 답변 가능, 없으면 스킵 허용(anti-fabrication). *트레이드오프: 순증 질문이 2→3개가 됨. 저마찰 원칙과의 균형은 사용자 최종 판단.*
2. **Q2 신규 후속 문구 수정 (S12 흡수)**: "그 방법을 고른 이유가 있나요?" → `"그 방법(또는 그 AI 도구)을 고른 이유가 있나요? 고려했다 접은 다른 방법/도구가 있으면 한 문장으로."` — 질문 수 그대로, 도구 선택 근거까지 포괄.
3. **Q5 후속 문구 보강 (O12 흡수)**: 기존 "정량 수치+스크린샷/링크?" 뒤에 `"이 결과물, 지금도 쓰고 계세요? (사용 빈도/기간)"` 옵션 추가 — 같은 턴에서 자연스럽게 이어 묻는 수준.

**편집 5 (자가진단 게이트) 델타:**
- 게이트 항목 [9] 신규: `[9] 본문에 회사명·API 키·개인정보·내부 URL 등 민감정보 없음 (마스킹/일반화 확인)` — X1 승격분. 미충족 시 해당 부분 마스킹 후 재확인 (질문 불요, 자체 수정).
- 게이트 [2](정량 수치) 판정 기준에 O14 추가: *"수치가 요약·본문·표에서 서로 일치하는지 교차 확인"*.
- 게이트 판정 전 생성 규칙에 S16 추가: *"DEVLOG 날짜 범위에서 총 제작 기간을 산출해 본문에 반영"* (자동, 질문 불요).

**최종 마찰 요약**: 76개 항목 중 사용자에게 새 질문을 요구하는 것은 최대 **3개**(확정 2 + 권장 1), 나머지 73개는 전부 기존 질문 커버 / 문구 보강 / 게이트 판정 기준 / 생성 규칙 / 패시브 확인으로 흡수. 저마찰 원칙(Principle 4)과 anti-fabrication(Principle 5)은 그대로 유지되며, X1 게이트는 오히려 원본 스킬의 보안 경고(376줄)를 Phase 3까지 확장해 정합성을 높인다.
