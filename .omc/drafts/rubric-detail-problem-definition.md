# 문제정의 (Problem Definition) — 20점 세부 채점 항목 설계

> write-post(goksung 모드)가 생성하는 `AI_CASE_STUDY.md`를 사람 채점자가 100점 루브릭으로 평가할 때,
> **문제정의(20점)** 카테고리에서 실제로 훑어보는 세부 기준을 "그물식(net)"으로 촘촘히 나열한 문서.
> 각 항목은 4부 구조(항목명 / 채점자가 이걸 볼 이유 / 충족기준 / write-post 연결점)로 기술한다.
>
> 대상 채점자 페르소나: 사내 AI 도입 프로그램 심사자, 부트캠프 코호트 과제 평가자, 커뮤니티 어워즈 심사위원.
> 이들이 "AI 활용 사례"에서 문제정의를 볼 때의 공통 관심사를 기준으로 설계했다.

---

## 세부 채점 항목 (12개)

### 1. 문제의 구체성 (Concreteness / Specificity)
- **왜 채점자가 이걸 볼 것 같은지**: 채점자는 "업무가 비효율적이었다" 류의 추상적 서술을 가장 흔한 감점 사유로 본다. 구체적 상황이 있어야 글쓴이가 진짜 그 문제를 겪었는지 판단할 수 있고, 뒤따르는 정량화·해결책의 신뢰도가 여기서 결정된다. 문제정의의 뼈대다.
- **충족기준**:
  - 충분함 = 특정 업무명·도구·반복 작업·산출물이 이름으로 명시됨 (예: "매주 월요일 5개 채널 실적을 손으로 취합해 보고서 표 만들기").
  - 부족함 = "여러 업무", "각종 자료 정리" 처럼 추상 명사만 있고 무엇을 하는지 그림이 안 그려짐.
  - 없음 = 문제 상황 서술 자체가 없거나 솔루션 자랑으로 바로 시작함.
- **write-post 연결점**: 😫 문제상황(Before) 섹션 / Phase 3 코어 질문(문제상황+시작계기). **기존 코어 질문으로 이미 능동 유도됨.** 새 후속질문 불필요.

### 2. 정량적 심각도 (Quantified Severity)
- **왜 채점자가 이걸 볼 것 같은지**: 시간·비용·빈도 손실이 숫자로 나와야 문제의 "크기"를 객관적으로 잴 수 있다. 숫자는 나중 성과(After) 대비 효과를 계산하는 기준점이 되므로, 정량화가 없으면 20점 만점 구간으로 올라가지 못한다. 채점자가 상·중·하를 가르는 1순위 신호.
- **충족기준**:
  - 충분함 = 구체 수치 1개 이상 (주 N시간 / 월 M만원 / 건당 X분 / 월 Y건 반복 등).
  - 부족함 = "시간이 많이 들었다", "번거로웠다" 같은 정성 표현만.
  - 없음 = 심각도/부담에 대한 언급 자체가 없음.
- **write-post 연결점**: 😫 문제상황(Before) 섹션. **기존 합의안의 goksung 후속질문("그 불편함을 숫자로 표현하면?")이 정확히 이 항목을 커버함.** → 그룹 A 유지.

### 3. 시작 계기의 진정성 / 결정적 순간 (Origin Authenticity / Last Straw)
- **왜 채점자가 이걸 볼 것 같은지**: "왜 하필 그때 시작했나"에 대한 구체적 트리거(마지막 지푸라기)는 실제 겪은 사람만 쓸 수 있는 디테일이다. 채점자는 이걸로 사례의 진위(가짜/과장 여부)를 감별한다. 계기가 생생하면 스토리 몰입도와 신뢰도가 동반 상승한다.
- **충족기준**:
  - 충분함 = 특정 사건·시점·감정이 명시됨 (예: "분기 마감날 밤 11시에 같은 표를 세 번째 다시 만들다가").
  - 부족함 = "그냥 불편해서", "효율화하고 싶어서" 같은 일반론.
  - 없음 = 시작 계기 언급 없음.
- **write-post 연결점**: Phase 3 코어 질문이 "시작 계기 / 더 이상 미룰 수 없었던 이유"를 이미 묻는다. **기존 코어 질문으로 능동 유도됨.** 다만 답변이 자주 얇게 나오므로, 코어 질문 프롬프트에 "결정적 순간" 예시를 강화하는 선에서 보강(신규 후속질문은 불필요 — 아래 그룹 논의 참고).

### 4. 긴급성 / 필요성 (Urgency / Necessity)
- **왜 채점자가 이걸 볼 것 같은지**: "안 하면 어떤 손실이 계속되는가"가 드러나야 문제 해결의 정당성이 선다. 채점자는 "있으면 좋은(nice-to-have)" 문제와 "안 하면 안 되는(must-fix)" 문제를 구분하려 한다. 후자일수록 고득점.
- **충족기준**:
  - 충분함 = 미루면 누적되는 손실/리스크/데드라인이 명시됨.
  - 부족함 = 필요성이 암시만 되고 명시 안 됨.
  - 없음 = 그냥 "해보고 싶었다" 수준.
- **write-post 연결점**: Phase 3 코어 질문의 "더 이상 미룰 수 없었던 이유" 하위 항목. **기존 코어 질문 커버.** 별도 후속질문 불필요 — 수동 확인.

### 5. 문제의 범위 (Scope: 개인 / 팀 / 조직)
- **왜 채점자가 이걸 볼 것 같은지**: 특히 사내 AI 도입 심사자는 "이 문제가 나 혼자 것인가, 팀/조직 공통인가"로 임팩트를 가늠한다. 범위가 넓을수록 확산 가치가 크다고 본다. 개인 문제여도 감점은 아니지만, 범위 명시 여부가 성숙도 신호.
- **충족기준**:
  - 충분함 = 영향 범위가 명시됨 (나 개인 / 우리 팀 N명 / 전사 등).
  - 부족함 = 범위가 불분명해 독자가 추측해야 함.
  - 없음 = 범위에 대한 단서 전혀 없음.
- **write-post 연결점**: 😫 문제상황(Before) + 🎯 이런 분들께 도움돼요에서 부분적으로 드러남. 신규 질문 없음 — **수동 확인(그룹 B).** 인터뷰를 늘리기보다 기존 답변에서 추론.

### 6. 독자 관련성 / 보편성 (Reader Relatability / Universality)
- **왜 채점자가 이걸 볼 것 같은지**: 커뮤니티/코호트 심사자는 "다른 사람에게도 도움이 되는 글인가"를 본다. 너무 특수한 문제는 공감을 못 얻고, 지나치게 일반적이면 구체성이 죽는다. 문제와 타깃 독자가 논리적으로 연결돼야 한다.
- **충족기준**:
  - 충분함 = 🎯 타깃 독자군이 명시되고, 그 독자가 겪을 법한 문제로 서술됨(문제-독자 정합).
  - 부족함 = 타깃이 "누구나" 수준으로 뭉뚱그려짐.
  - 없음 = 타깃 독자 언급 없음.
- **write-post 연결점**: 🎯 이런 분들께 도움돼요 섹션(전용 질문 존재). **기존 섹션 커버.** 신규 후속질문 불필요 — 수동 확인.

### 7. 문제-해결 논리 연결 (Problem–Solution Logical Fit)
- **왜 채점자가 이걸 볼 것 같은지**: 채점자가 은근히 가장 중시하는 축. "서술한 문제가 실제로 만든 것을 정당화하는가?" 문제는 A(취합 시간)인데 만든 건 B(디자인 자동화)면 문제정의가 붕 뜬다. 글 전체의 신뢰를 좌우하므로 문제정의 점수의 상한을 결정한다.
- **충족기준**:
  - 충분함 = Before에 적힌 핵심 pain point가 솔루션 섹션의 기능과 1:1로 대응됨.
  - 부족함 = 문제와 솔루션이 대략 관련 있으나 일부 pain은 해결 안 됨/일부 기능은 근거 없음.
  - 없음 = 문제와 만든 것이 서로 다른 얘기.
- **write-post 연결점**: 😫 문제상황(Before) ↔ 솔루션/결과 섹션 **교차 검증**. 인터뷰 질문으로 직접 묻기 어색한(글 전체 정합성) 항목이라 **작성 후 수동 일관성 체크(그룹 B)**로 두는 게 맞음.

### 8. 감정적 / 맥락적 생생함 (Emotional & Contextual Vividness)
- **왜 채점자가 이걸 볼 것 같은지**: 실제 겪은 사람만 나오는 구체적 장면·감정 묘사는 진정성의 방증이다. 채점자는 교과서적 서술보다 생생한 서술에 더 높은 몰입/신뢰 점수를 준다. 단, 정량화·구체성이 충족된 뒤의 가산 요소.
- **충족기준**:
  - 충분함 = 구체적 장면/감정 묘사 1개 이상("퇴근 30분 전 수정 요청에 한숨").
  - 부족함 = 사실 나열만 있고 톤이 건조.
  - 없음 = 무관(생생함 부재는 감점보다 무가점).
- **write-post 연결점**: 😫 문제상황(Before) 섹션. 인터뷰를 더 늘리지 않고 **수동 확인(그룹 B)** — 글쓰기 단계 톤 가이드로 흡수.

### 9. 문제의 반복성 / 지속성 (Recurrence / Chronic Nature)
- **왜 채점자가 이걸 볼 것 같은지**: 일회성 불편보다 반복되는 만성 문제일수록 자동화/AI 적용 가치가 크다. 채점자는 "매번" "매주" 같은 빈도 신호로 문제의 무게를 재확인한다.
- **충족기준**:
  - 충분함 = 빈도 표현 명시(매일/매주/매 프로젝트마다 등).
  - 부족함 = 반복 여부 불명확.
  - 없음 = 빈도 언급 없음.
- **write-post 연결점**: 정량적 심각도(#2)와 겹침 — "숫자로 표현하면?" 후속질문 답변에 빈도가 자연히 포함되는 경우가 많음. **별도 질문 불필요 — 수동 확인(그룹 B).**

### 10. 기존 대안의 부재 / 실패 (Why Existing Solutions Didn't Work)
- **왜 채점자가 이걸 볼 것 같은지**: 고득점 사례를 가르는 결정적 축. "기존 툴/방법으로 왜 안 됐나"가 있으면 (a)문제가 사소하지 않다는 증거, (b)AI/직접 제작을 택한 정당성이 동시에 서고, (c)글쓴이의 문제 이해 깊이가 드러난다. 이 항목이 비어 있으면 채점자는 "굳이 만들 필요 있었나?"라는 의심을 품는다. **현재 write-post 어떤 질문으로도 커버되지 않는 사각지대.**
- **충족기준**:
  - 충분함 = 시도했던 기존 방법/툴과 그 한계가 서술됨(예: "엑셀 매크로로 해봤지만 채널 포맷이 매번 바뀌어 유지보수가 더 오래 걸렸다").
  - 부족함 = 대안 언급은 있으나 왜 부족했는지 없음.
  - 없음 = 기존 시도/대안 언급 전혀 없음.
- **write-post 연결점**: **연결된 기존 질문/섹션 없음.** Phase 3에 **신규 goksung 후속질문 1개 추가를 권장** → 그룹 A. (제안 문구는 아래 그룹 정리 참고.)

### 11. 성과 측정 연결성 (Measurability / Implied Success Criteria)
- **왜 채점자가 이걸 볼 것 같은지**: 문제정의 단계에서 제시한 수치가 나중 성과(After) 측정의 기준이 되는지 본다. 문제의 정량 지표와 결과 지표가 같은 단위로 대응되면 글의 완결성이 크게 오른다.
- **충족기준**:
  - 충분함 = 문제 단계 수치와 성과 단계 수치가 같은 축으로 대응 가능(before N시간 → after M시간).
  - 부족함 = 문제 수치는 있으나 성과와 단위가 안 맞음.
  - 없음 = 문제에 측정 가능한 지표 없음.
- **write-post 연결점**: #2(정량 심각도)의 파생 — 정량 후속질문이 충족되면 자동 확보. 성과 섹션과의 정합은 **작성 후 수동 교차 확인(그룹 B).**

### 12. 대상 독자의 구체성 (Target Reader Clarity)
- **왜 채점자가 이걸 볼 것 같은지**: #6의 독자 관련성이 "문제↔독자 정합"이라면, 이 항목은 독자 페르소나 자체의 선명도다. "이 글이 정확히 누구를 위한 것인지"가 뚜렷하면 심사자는 실용성/전달력에 가점을 준다.
- **충족기준**:
  - 충분함 = 구체 페르소나 명시(직무/상황/도구 수준으로).
  - 부족함 = "직장인 누구나" 수준.
  - 없음 = 타깃 독자 언급 없음.
- **write-post 연결점**: 🎯 이런 분들께 도움돼요 섹션(전용 질문 존재). **기존 커버.** 신규 질문 불필요 — 수동 확인.

## 필수(A) / 선택(B) 그룹 분류

> 설계 원칙: write-post는 "한 번에 질문 하나"의 저마찰 UX가 생명이다. Phase 3 문제정의 구간은 이미 **코어 질문 1개(문제상황+시작계기)** 를 던지고 있고, 합의안이 **goksung 후속질문 1개("숫자로 표현하면?")** 를 추가한다. 여기에 후속질문을 더 얹을 때는 "채점 배점을 실제로 끌어올리면서 기존 질문으로 커버 안 되는 것"만 엄선한다. 아래는 최대 1~2개 신규 후속질문만 A로 승격한 결과다.

### (A) 필수 확인 질문 — goksung 모드가 없으면 능동적으로 후속질문
| # | 항목 | 커버 경로 | 조치 |
|---|------|-----------|------|
| 1 | 문제의 구체성 | Phase 3 코어 질문 | **기존 코어 질문으로 이미 능동 유도** (신규 불필요) |
| 3 | 시작 계기의 진정성 | Phase 3 코어 질문 | **기존 코어 질문으로 능동 유도** (신규 불필요, 프롬프트 예시만 보강 권장) |
| 4 | 긴급성/필요성 | Phase 3 코어 질문 | **기존 코어 질문으로 능동 유도** (신규 불필요) |
| 2 | 정량적 심각도 | goksung 후속질문(합의안) | **"그 불편함을 숫자로 표현하면?" 유지** ✅ |
| 10 | 기존 대안의 부재/실패 | **커버 안 됨 (사각지대)** | **신규 goksung 후속질문 1개 추가 권장** ⭐ |

**신규 후속질문 제안 (1개만):**
> "혹시 이 문제를 그동안 다른 방법(엑셀, 기존 툴, 수작업 등)으로 해결해보려 한 적 있나요? 그 방법으로는 왜 부족했나요?"

- **근거**: #10은 (a) 배점 상단(고득점 사례 감별)에 직결되고, (b) Phase 3 어떤 기존 질문으로도 안 잡히는 유일한 큰 사각지대이며, (c) 답변이 문제의 심각도·정당성·글쓴이 이해 깊이를 한 번에 채운다. 후속질문 1개 추가 대비 회수 배점이 가장 크다.
- **판단**: 합의안의 정량 후속질문 1개만으로는 **부족**하다고 본다. #10을 더한 **총 2개의 goksung 후속질문**이 문제정의 20점을 저마찰로 방어하는 최적점이다. #3(결정적 순간)은 이미 코어 질문에 포함되므로 신규 질문으로 중복시키지 않고, 코어 질문 프롬프트의 예시 문구 강화로만 처리한다.
