# 해결과정 (Solution Process) — 20점 세부 채점 항목 그물식 설계

> 대상 섹션: 게시글의 **🔧 작업 과정** (스토리텔링형, 에피소드 반복 구조 — 사용자 원문 요청 코드블록 + AI가 한 일 + 결과/인사이트 서술)
> 표준 게이트(합의안 기준): 에피소드 ≥2, 각각 "사용자 원문 코드블록 + 막힘→해결 비트"
> 관련 Phase 3 질문: Q2(작업 하이라이트), Q3(인상적인 순간), Q4(막혔던 부분)
> 이 문서의 목적: 채점자가 20점을 매길 때 실제로 훑는 세부 항목을 그물식으로 나열하고, 각각을 write-post 인터뷰 흐름에 매핑한다.

---

## 채점 세부 항목 (11개)

### 1. 접근법의 논리적 타당성
- **왜 채점자가 이걸 볼 것 같은지**: "AI 썼다"가 아니라 "문제를 어떻게 풀었나"가 이 카테고리의 본질. 채점자는 선택한 경로가 문제 정의에서 자연스럽게 도출되는지, 아니면 뜬금없는 삽질인지를 먼저 본다. 접근이 문제와 논리적으로 연결돼 있어야 나머지 항목이 의미를 가진다.
- **충족기준**:
  - 충분: 문제 → 접근법 사이에 "왜 이렇게 접근했는가"가 한 문장이라도 드러남 (예: "수작업이 반복돼서 우선 파싱을 자동화하기로 했다").
  - 부족: 무엇을 했는지는 있으나 왜 그 순서/그 방향인지 인과가 비어 있음.
  - 없음: 툴을 나열만 하고 문제와의 연결이 전혀 없음.
- **write-post 연결점**: Q2(작업 하이라이트)에 이미 소재가 있음. 별도 후속질문 불필요 — 하이라이트를 서술할 때 자연스럽게 "왜 그걸 먼저 했나"가 나오면 충족. **패시브 확인.**

### 2. 대안 검토 — 왜 다른 방법이 아니라 이 방법인가
- **왜 채점자가 이걸 볼 것 같은지**: 초보와 숙련자를 가르는 핵심 신호. "이 방법밖에 몰라서" 쓴 것과 "A/B/C를 저울질하고 골랐다"는 것은 의사결정 성숙도가 완전히 다르다. 채점자는 대안 언급 유무로 사고의 깊이를 즉시 판별한다.
- **충족기준**:
  - 충분: 최소 1개의 고려했던 다른 방법과 기각 사유가 언급됨 (예: "직접 크롤링도 고려했지만 유지보수가 커서 API로 갔다").
  - 부족: "여러 방법이 있었다"는 추상적 언급만 있고 구체적 비교가 없음.
  - 없음: 대안에 대한 언급이 전혀 없어 마치 유일한 선택지였던 것처럼 읽힘.
- **write-post 연결점**: **현재 커버 안 됨.** Q2/Q4 어디에도 "왜 이 방법이었나"를 끌어내는 장치가 없다. → **그룹 A 후보 (신규 후속질문 필요).**

### 3. 시행착오의 구체성
- **왜 채점자가 이걸 볼 것 같은지**: "몇 가지 이슈가 있었다" 같은 뭉뚱그린 서술은 실제로 겪지 않았거나 복기하지 않았다는 신호로 읽힌다. 채점자는 "무엇이, 왜 실패했는가"의 해상도를 본다 — 구체적 실패담은 진정성과 학습의 증거다.
- **충족기준**:
  - 충분: 특정 실패가 원인과 함께 서술됨 (예: "AI가 날짜 포맷을 ISO로 넣어서 정렬이 깨졌다 → 로케일을 명시").
  - 부족: 실패가 있었다는 사실만 있고 원인·증상이 불명확.
  - 없음: 모든 게 매끄럽게 됐다는 식으로 시행착오가 아예 부재.
- **write-post 연결점**: Q4(막혔던 부분)가 직접 소재. 합의안의 Q4 후속질문("그 막힘을 어떻게 해결했나요?")과 결합해 구체화됨. **커버됨 (Q4 + 기존 후속).**

### 4. 의사결정 근거의 질
- **왜 채점자가 이걸 볼 것 같은지**: 갈림길에서의 판단이 "감"인지 "근거"인지를 본다. 근거가 관찰/제약/목표에 기반하면 재현·신뢰 가능하고, 없으면 운으로 읽힌다.
- **충족기준**:
  - 충분: 최소 1개의 결정에 대해 "무엇을 보고 그렇게 정했는지" 판단 기준이 드러남.
  - 부족: 결정은 있으나 근거가 "그게 나아 보여서" 수준.
  - 없음: 결정 지점 자체가 서술되지 않음.
- **write-post 연결점**: Q2·Q4에 걸쳐 분산 소재. 항목 2(대안 검토)의 후속질문이 채워지면 이 항목도 상당 부분 함께 충족됨 — 별도 질문 불요. **패시브 확인 (항목 2에 종속).**

### 5. AI 도구 활용의 구체성 — 수동 사용 vs 능동 협업
- **왜 채점자가 이걸 볼 것 같은지**: "AI에게 시켰다"(수동)와 "AI 출력을 판단하고 되받아쳐 방향을 잡았다"(능동 협업)는 역량 신호가 다르다. 채점자는 사용자가 운전석에 있었는지, 조수석에서 받아쓰기만 했는지를 본다.
- **충족기준**:
  - 충분: 사용자가 AI 출력에 개입·판단·수정을 가한 장면이 최소 1회 구체적으로 드러남.
  - 부족: AI가 다 해줬다는 서술뿐, 사용자의 판단 개입이 안 보임.
  - 없음: 프롬프트 한 번 = 결과물, 협업 과정이 소거됨.
- **write-post 연결점**: Q3(인상적인 순간) + Q4 해결 서술에 소재 존재. Q3는 합의안에서 변경 없음(패시브). 능동 협업 장면은 대개 Q3/Q4 답변에 자연 노출되므로 **패시브 확인.**

### 6. 반복/개선 사이클의 존재
- **왜 채점자가 이걸 볼 것 같은지**: 한 번에 완성됐다는 서사는 비현실적이거나 난이도가 낮았다는 신호. 채점자는 "고치고-다시-고친" 루프의 흔적을 실전성의 증거로 본다.
- **충족기준**:
  - 충분: 최소 1개 에피소드에서 "1차 결과 → 문제 발견 → 2차 개선"의 순환이 드러남.
  - 부족: 개선이 있었다는 암시는 있으나 사이클로 서술되지 않음.
  - 없음: 모든 산출이 단발성으로 제시됨.
- **write-post 연결점**: Q4(막힘→해결) 후속질문이 사실상 1회 사이클을 강제함. 에피소드 구조 자체가 반복 비트를 담을 수 있음. **커버됨 (Q4 + 에피소드 템플릿).**

### 7. 난이도 대비 해법의 우아함
- **왜 채점자가 이걸 볼 것 같은지**: 어려운 문제를 간결하게 풀었는지, 쉬운 문제를 과하게 부풀렸는지를 본다. 채점자는 문제의 기술적 난이도와 해법의 경제성 사이 비율로 인상 점수를 조정한다.
- **충족기준**:
  - 충분: 문제의 어려움이 드러나고 해법이 그에 비해 간결/영리함 (또는 정직하게 "단순 문제였다"고 밝힘).
  - 부족: 난이도 맥락 없이 해법만 있어 우아함을 판단할 수 없음.
  - 없음: 쉬운 문제를 불필요하게 복잡한 서사로 포장 (역효과).
- **write-post 연결점**: Q2 하이라이트 선택 자체가 난이도 신호를 담음. 별도 질문은 친다면 과함. **패시브 확인.**

### 8. 재현 가능성
- **왜 채점자가 이걸 볼 것 같은지**: 사례글의 가치는 "독자가 따라 할 수 있는가"에서 나온다(업무적용 카테고리와도 연동). 사용자 원문 요청 코드블록이 존재하는 이유가 바로 이것 — 실제 프롬프트가 있어야 재현된다.
- **충족기준**:
  - 충분: 사용자 원문 요청(코드블록)이 실제 재사용 가능한 구체성으로 최소 2개 에피소드에 존재.
  - 부족: 원문이 있으나 너무 추상적("잘 만들어줘")이라 따라 할 단서가 안 됨.
  - 없음: 원문 코드블록 부재 — 표준 게이트 미달.
- **write-post 연결점**: 에피소드 템플릿(사용자 원문 코드블록)이 구조적으로 강제. Q2 소재로 채워짐. **커버됨 (템플릿 게이트).**

### 9. 막힘 원인 분류의 정직성 (AI 오해 vs 내가 뭘 물을지 몰랐음)
- **왜 채점자가 이걸 볼 것 같은지**: Q4가 명시적으로 구분을 요구하는 지점. "AI가 잘못 이해함"과 "내가 요청을 잘 못함"은 성찰의 방향이 정반대다. 후자를 정직하게 인정하는 글은 메타인지 신호로 높게 평가된다 — 채점자는 남 탓/자기성찰의 균형을 본다.
- **충족기준**:
  - 충분: 막힘의 원인이 두 유형 중 어느 쪽인지 정직하게 명시되고, 특히 "내가 요청을 다듬었다"는 자기교정이 있으면 가점.
  - 부족: 막힘은 언급되나 원인이 "AI가 이상했다"로만 귀결.
  - 없음: 막힘 서술 자체가 없음 (표준 게이트 미달 위험).
- **write-post 연결점**: Q4(막혔던 부분)가 이 구분을 이미 질문 문면에 담고 있음. 합의안 후속("어떻게 해결했나요")과 결합. **커버됨 (Q4).**

### 10. 인간–AI 역할 분담의 명료성
- **왜 채점자가 이걸 볼 것 같은지**: 무엇을 사람이 판단하고 무엇을 AI가 실행했는지 경계가 흐리면, 성과의 귀속이 모호해지고 재현성이 떨어진다. 채점자는 "사용자의 기여가 어디인가"를 찾는다.
- **충족기준**:
  - 충분: 최소 한 에피소드에서 사람의 판단 몫과 AI의 실행 몫이 구분되어 읽힘.
  - 부족: "같이 했다" 수준으로 경계가 뭉개짐.
  - 없음: 전부 AI가 한 것처럼 서술되어 사용자 기여가 소거됨.
- **write-post 연결점**: 항목 5(능동 협업)와 강하게 겹침 — Q3/Q4 서술에서 함께 노출. 독립 질문 불요. **패시브 확인 (항목 5에 종속).**

### 11. 에피소드 서사의 인과 연결성 (요청→행동→결과→인사이트)
- **왜 채점자가 이걸 볼 것 같은지**: 각 에피소드가 "원문 요청 → AI가 한 일 → 결과/인사이트"로 인과가 이어져야 이야기가 학습으로 남는다. 나열형(요청만, 또는 결과만)은 점수가 정체된다.
- **충족기준**:
  - 충분: ≥2 에피소드가 4단계 인과(요청→행동→결과→그래서 뭘 배웠나)를 모두 갖춤.
  - 부족: 요청·행동은 있으나 "그래서 뭐가 달라졌나/뭘 배웠나"가 빠짐.
  - 없음: 코드블록 나열 수준으로 서사가 끊김.
- **write-post 연결점**: 에피소드 템플릿 구조 + Q2 소재로 대부분 충족. 인사이트 꼬리는 Q4 해결 서술과 연동. **커버됨 (템플릿 + Q2/Q4).**

## (A) 필수 확인 질문 — 없으면 goksung 모드에서 능동 후속질문 정당화

**현행 합의안 점검**: Q2(작업 하이라이트) + Q4(막혔던 부분) + Q4 후속("그 막힘을 어떻게 해결했나요?"). 이 조합은 위 11개 중 **항목 1, 3, 4(부분), 5, 6, 8, 9, 10, 11**을 이미 커버한다. 즉 시행착오·반복·재현·역할분담·정직한 원인분류·인과서사는 기존 흐름만으로 채워진다.

**유일하게 구조적으로 비어 있는 고가치 항목 = 항목 2 (대안 검토 / 왜 이 방법인가)** 그리고 그에 종속된 항목 4(의사결정 근거의 질). 이 둘은 "왜 다른 방법이 아니라 이 방법인가"라는 단 하나의 질문으로 동시에 채워진다. 초보/숙련을 가르는 최상위 신호이면서 현행 인터뷰의 사각지대이므로, 원-퀘스천 추가 비용이 정당화된다.

- **추가 권장 1개 (Q2에 부착하는 조건부 후속)**:
  > "그 방법을 고른 이유가 있나요? 혹시 고려했다 접은 다른 방법이 있다면 한 문장으로." (한 문장, 없으면 스킵 가능)

  - 근거: 항목 2 + 항목 4를 한 번에 충족. 20점 중 의사결정 성숙도 신호를 직접 확보. Q4 후속과 마찬가지로 "한 문장" 저마찰 형식이라 one-question-at-a-time UX를 깨지 않음. "없으면 스킵"으로 anti-fabrication 원칙 유지(억지 대안 창작 방지).

**결론**: 그룹 A는 **위 1개만 추가**를 권장한다. 그 이상은 저마찰 인터뷰 UX를 해치고 한계효용이 급감한다. (합의안의 Q4 후속 1개 + 본 추가 1개 = 해결과정 카테고리 전용 후속질문 총 2개.)

## (B) 선택적 심화 확인 — 패시브 체크 전용 (추가 질문 불요)

아래 항목은 기존 답변(Q2/Q3/Q4)과 에피소드 템플릿 구조에서 자연 노출되므로, 별도 인터뷰 질문 없이 **자가진단 게이트/작성 단계에서 존재 여부만 패시브로 확인**한다.

| 항목 | 패시브 커버 경로 |
|---|---|
| 1. 접근법의 논리적 타당성 | Q2 하이라이트 서술에 내재 |
| 4. 의사결정 근거의 질 | 그룹 A 추가 질문에 종속 충족 |
| 5. AI 도구 활용의 구체성(능동 협업) | Q3 + Q4 해결 서술 |

**패시브 확인 시 게이트 문구 예시** (작성 단계 자가진단용, 사용자에게 질문하지 않음):
